# commandbase-vault

Vault BRDSPI workflow for Obsidian vault management — brainstorm, research, design, structure, plan, implement chain. Requires commandbase-core for docs agents.

## Dependencies

- commandbase-core (docs agents)

## Skills

| Skill | Description |
|-------|-------------|
| /brainstorming-vault | Explore direction and preferences for an Obsidian vault before building structure |
| /designing-vault | Make organizational decisions: MOC strategy, tagging taxonomy, template patterns |
| /implementing-vault | Execute vault implementation plans — create notes, apply frontmatter, manage structure |
| /importing-vault | Convert .docs/ artifacts into Obsidian vault notes with wikilinks and callouts |
| /planning-vault | Create or iterate on vault implementation plans with thorough vault research |
| /researching-vault | Research vault structure, conventions, link graphs, and content patterns |
| /starting-vault | Initialize a new Obsidian vault or configure Claude Code to work with an existing one |
| /structuring-vault | Map folder layout, naming conventions, and note placement rules |

## Installation

```shell
/plugin install commandbase-vault
```
